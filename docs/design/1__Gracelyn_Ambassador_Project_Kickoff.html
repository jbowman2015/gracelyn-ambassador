3
Open Wise Business and Tremendous accounts You
These two accounts have a 1-3 day business verification process. Open them immediately — do not wait. Everything else in the build can proceed in parallel while verification completes. Both are free to open.
  • Wise Business — wise.com/business. Fund with an initial balance for outgoing international transfers.
  • Tremendous — tremendous.com. Pay-as-you-go, no monthly fee. API key available immediately after signup.
  • Confirm PayPal Business account is active with Payouts API enabled.
4
Build CRM modules and fields in Zoho You
This is the longest UI task in the pre-build phase. Work through the Pre-Build Checklist v2.0 Section 1 in order. Three modules need to be built or extended.
  • Ambassadors module — 39 new fields added to the existing module. Work from the checklist field-by-field. Encrypted field type required for Bank_Account_Routing and Bank_Account_Number.
  • Prospects module — entire new module, 20 fields. Create this before Agent 0 development begins.
  • Support Tickets module — entire new module, 12 fields. Critical: the exact API names here must match what is in the Agent 4 and Agent 5 documents. Do not rename fields.
Record every field API name exactly as you create it. Developers reference these names in every CRM call.
5
Create WorkDrive folder structure and upload content You
Nine folders required. Most already exist. What is new in v2.0:
  • Folder 06 subfolders — create four subfolders: k12-educator, early-childhood, faith-community, youth-serving. Upload one lead magnet file to each before Agent 1D is activated.
  • Folder 08 — upload the four brand asset text files produced in Steps 1 and 2.
  • Folder 09 — create this folder for VIP Prospect briefings. Agent 0 writes here. Agent 1B reads here. VIP Relationship Manager receives links from here.
Record every folder ID. Developers need these IDs before they can write a single WorkDrive API call.
6
Build all 8 Zoho Forms You
Field names and conditional logic for all 8 forms are specified in the Agent 2 and Agent 1D documents. The two most critical are new in v2.0.
  • Combined compliance form — three scrollable sections (Agreement, Ethics, Training Guide), three e-signature capture areas, motivation discovery questions before Part 3 signature, single submission. Replaces the three sequential documents from v1.
  • Win-back survey form — one question, four radio button options (Too Busy, Confused, Not Sure, Technical Problem). Linked from the Day 15 win-back email. Response fires Make.com routing.
  • Story intake form — batch submission, up to 5 stories per submission, Role Category dropdown required on each slot.
  • Four lead capture forms — one per audience track with hidden lead_magnet_id, utm_source, and utm_campaign fields.
Record every form ID. Agent 2 and Agent 1D reference these.

8
Confirm all policy environment variables with Dr. Flippen You
Twenty policy decisions are locked and documented in the Pre-Build Checklist v2.0 Section 8. These become Catalyst environment variables. Before any agent code runs against a live system, Dr. Flippen should confirm the five most operationally sensitive values.
  • APPROVAL_MODE — starts as MANUAL. Dr. Flippen decides when to flip to AUTO.
  • VIP percentage bands — 2.5% / 5% under 10k ambassadors, 0.5% / 2.5% above. Confirmed.
  • NON_REFERRAL_DAYS_THRESHOLD — 90 days. Confirmed.
  • DORMANT_DAYS_THRESHOLD — 30 days. Confirmed.
  • MISSION_KEYWORDS — initial list for Agent 1B warm-follow matching. Needs Dr. Flippen input.
9
WordPress portal changes — schedule with Parmeet Parmeet
Claude Code produces the WordPress changes needed (role types, referral dashboard display logic, portal chat widget integration). Do not deploy to WordPress without Parmeet present. Schedule a working session where you both review the changes together before anything is applied to the live site. This is the one task in the build where the conservative approach is non-negotiable — a broken WordPress site affects current students.
10
Run Agent 5A validation — go/no-go gate Both
When all pre-build tasks are complete and all Catalyst environment variables are set, Agent 5A runs. It checks every credential and environment variable before any production agent is deployed. If 5A passes, the build proceeds. If it fails, it tells you exactly which variable is missing or invalid. Nothing deploys until 5A gives a clean pass.
Three cross-agent coordination points that must be confirmed at kickoff between Claude Code sessions: (1) Agent 3 and Agent 6 must use the exact same ROLE_CATEGORY header line label in story files — a mismatch produces silent fallback with no error. (2) Agent 4 and Agent 5 must use the exact same API names for all 9 Support Tickets SLA fields — a mismatch breaks SLA tracking silently. (3) The escalation_timestamp in Agent 5's webhook payload must be the identical ISO string written to Escalation_Timestamp in the CRM ticket — Agent 4's SLA calculation depends on this match.
